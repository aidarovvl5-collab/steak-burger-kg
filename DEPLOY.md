# STEAK BURGER: загрузка на GitHub и Render

## Какие файлы загрузить в GitHub

Загружайте все файлы в корень репозитория, без папок:

- `index.html`
- `admin.html`
- `server.js`
- `package.json`
- `store.json`
- `.gitignore`
- `DEPLOY.md`

Папки `admin` и `data` больше загружать не нужно. Я оставил их на ноутбуке как резервную копию, но для GitHub удобнее использовать файлы выше.

## Как загрузить через сайт GitHub

1. Откройте ваш репозиторий на GitHub.
2. Нажмите `Add file`.
3. Нажмите `Upload files`.
4. Перетащите файлы из списка выше.
5. Внизу напишите сообщение:

```text
Initial upload
```

6. Нажмите `Commit changes`.

Важно: не загружайте `steak-burger-upload.zip`. Render не распакует его как проект.

## Как запустить на Render

1. Откройте Render.
2. Нажмите `New`.
3. Выберите `Web Service`.
4. Подключите GitHub и выберите репозиторий.
5. Укажите настройки:

```text
Language: Node
Build Command: npm install
Start Command: npm start
Plan: Free
```

6. Нажмите `Deploy Web Service`.

После завершения Render даст ссылку на сайт. Админка будет открываться по адресу:

```text
https://ваш-сайт.onrender.com/admin
```

## Telegram для операторов

Токен бота нельзя загружать в GitHub. Его нужно добавить только в Render:

1. Откройте ваш сайт в Render.
2. Перейдите в `Environment`.
3. Добавьте переменные:

```text
TELEGRAM_BOT_TOKEN=токен_бота_из_BotFather
TELEGRAM_CHAT_ID=id_группы_заказов
TELEGRAM_WEBHOOK_SECRET=любой_длинный_секретный_текст
```

4. Нажмите `Save Changes`.
5. Дождитесь перезапуска сайта.

После этого нужно один раз включить webhook. Откройте в браузере ссылку, заменив значения на свои:

```text
https://api.telegram.org/botТВОЙ_ТОКЕН/setWebhook?url=https://ВАШ-САЙТ.onrender.com/api/telegram/webhook&secret_token=ТВОЙ_WEBHOOK_SECRET
```

Если Telegram ответил `ok: true`, бот подключен.

Как это работает:

- клиент оформляет заказ на сайте;
- заказ сохраняется в системе;
- в Telegram-группу приходит карточка заказа;
- оператор нажимает кнопку статуса;
- статус обновляется в системе и в админке управляющего.

## Загрузка фото из админки

Google Drive и Яндекс Диск обычно дают ссылку на страницу просмотра, а не на сам файл картинки. Поэтому для удобной загрузки фото из админки используйте Cloudinary.

1. Создайте аккаунт на Cloudinary.
2. В Cloudinary откройте `Settings` -> `Upload`.
3. Создайте `Upload preset`.
4. Сделайте preset `Unsigned`.
5. Желательно ограничить preset:
   - только изображения;
   - папка `steak-burger`;
   - максимальный размер файла;
   - форматы `jpg`, `png`, `webp`.
6. В Render откройте ваш сервис -> `Environment`.
7. Добавьте переменные:

```text
CLOUDINARY_CLOUD_NAME=ваш_cloud_name
CLOUDINARY_UPLOAD_PRESET=название_unsigned_preset
```

8. Нажмите `Save Changes` и дождитесь перезапуска сайта.

После этого в админке рядом с логотипом, фото блюда и баннером появится загрузка файла. Управляющий выбирает фото с компьютера, сайт загружает его в Cloudinary и сам вставляет ссылку в нужное поле.

## Важно про данные

Сейчас заказы, меню, клиенты и сотрудники хранятся в `store.json`. Для теста это нормально. Для реальной работы позже лучше подключить базу данных, чтобы заказы не терялись при обновлениях и перезапусках.
