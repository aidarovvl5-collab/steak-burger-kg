# Как тестировать STEAK BURGER на разных устройствах

## 1. Быстрый тест без хостинга

Подходит, если телефон и ноутбук подключены к одному Wi-Fi.

1. Запустите сайт на ноутбуке.
2. В PowerShell выполните:

```powershell
ipconfig
```

3. Найдите строку `IPv4 Address`, например `192.168.1.25`.
4. На телефоне откройте:

```text
http://192.168.1.25:4173
```

Если телефон не открывает сайт, Windows может блокировать входящие подключения. Тогда проще использовать тоннель или хостинг.

## 2. Быстрый публичный тест через тоннель

Это временная ссылка на сайт с вашего ноутбука. Удобно показать сайт другу или открыть с мобильного интернета.

Cloudflare Quick Tunnel:

```powershell
cloudflared tunnel --url http://localhost:4173
```

После запуска появится ссылка вида `https://....trycloudflare.com`.

Ngrok:

```powershell
ngrok http 4173
```

После запуска появится публичная HTTPS-ссылка.

## 3. Бесплатный хостинг для нашего проекта

Для текущего сайта лучше всего подходит Render, потому что у нас есть backend, админка и API заказов.

Важно: сейчас данные заказов, меню, клиенты и сотрудники хранятся в `data/store.json`. На бесплатных облачных хостингах такие файлы могут сбрасываться при перезапуске или повторной выкладке. Для теста это нормально. Для реальной работы позже лучше подключить базу данных.

### Подготовка GitHub

1. Создайте аккаунт на GitHub.
2. Создайте новый приватный репозиторий, например `steak-burger-bishkek`.
3. Загрузите в репозиторий файлы и папки проекта:
   - `index.html`
   - `server.js`
   - `package.json`
   - `.gitignore`
   - `admin/`
   - `data/`

Лучше делать репозиторий приватным, потому что это рабочий проект с админкой и данными.

### Выкладка на Render

1. Создайте аккаунт на Render.
2. Нажмите `New` -> `Web Service`.
3. Подключите GitHub и выберите репозиторий `steak-burger-bishkek`.
4. Настройки:
   - Runtime: `Node`
   - Build Command: можно оставить пустым или указать `npm install`
   - Start Command: `npm start`
   - Plan: `Free`
5. Нажмите `Deploy Web Service`.

Когда деплой закончится, Render даст ссылку вида:

```text
https://steak-burger-bishkek.onrender.com
```

Админка будет здесь:

```text
https://steak-burger-bishkek.onrender.com/admin
```

## 4. Что делать дальше

Для теста на разных телефонах достаточно Render или Cloudflare Tunnel.

Для настоящего запуска нужны еще 3 вещи:

1. Домен, например `steakburger.kg` или похожий.
2. База данных вместо `data/store.json`.
3. Подключение онлайн-оплаты, когда получите API.
